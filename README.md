# ProjectVault — Idea Index

Hobby project brainstorm and planning space. See [[CLAUDE]] for the full workflow.

---

## Quick Start

Open this folder in [Claude Code](https://claude.ai/code). On first use, Claude will ask 4 setup questions to build your personal profile (name, technical depth, stack preferences, idea capture style). That's it — you're ready.

| I want to… | Say… |
|------------|------|
| Capture a new idea | `new idea — [name]: [one sentence description]` |
| Dump several ideas at once | `idea: X`, `idea: Y`, `idea: Z` (or use `/intake`) |
| Design an existing seed idea | `/design [slug]` |
| Move an idea to a real repo | `/vault-init [slug]` |
| See all ideas and their status | `/vault-status` |
| Fix harness drift after updates | `/vault-resync` |

## Status Key

- **Seed** — just an idea, `Overview.md` only
- **Exploring** — actively researching stack and design
- **Designing** — architecture and phases being worked out
- **Building** — in active development, has a real repo
- **Parked** — paused indefinitely
- **Shipped** — deployed or "done enough"

## How to Start a New Idea

Say something like: `new idea — [name]: [one sentence description]`

Claude will run the full brainstorming workflow and produce:

```
ideas/[slug]/
├── Overview.md            ← created first, before anything else
├── Stack.md               ← after approach is chosen
├── Architecture.md        ← after design sections are approved
├── Phases.md              ← after scope is defined
├── Design-Spec.md         ← consolidated spec, source of truth for building
├── ProjectInitialize.md   ← generated by /vault-init [slug] when idea moves to a real repo
└── Research/
    ├── layout-types.html          ← core interaction model options
    ├── authoring-modes.html       ← creator editor options
    ├── learner-experience.html    ← learner perspective per layout
    ├── design-architecture.html   ← system architecture diagram
    ├── design-datamodel.html      ← database schema + JSONB shapes
    ├── design-frontend.html       ← component tree + route structure
    └── design-api-deploy.html     ← API endpoints + infra diagram
```

`README.md` is updated with a new row at the end of each session.

## Brainstorming Phase Flow

```
1. Clarify
   └── One question at a time (audience, scope, interaction model, constraints)
   └── Visual companion offered for questions involving mockups/diagrams

2. Propose Approaches
   └── 2–3 architectural options with trade-offs
   └── Recommendation + reasoning

3. Design Sections (browser mockups for each)
   └── Architecture overview
   └── Data model
   └── Frontend components + routes
   └── API endpoints + deployment

4. File Creation
   └── Overview → Stack → Architecture → Phases → Design-Spec
   └── Research/*.html — interactive artifacts saved as standalone files
   └── README.md updated

5. Implementation Plan (when ready to build)
   └── Spec splits into subsystems (backend / frontend / infra)
   └── One plan per subsystem, starting with backend
   └── ProjectInitialize.md created for repo handoff
```

## Moving an Idea to a Real Repo

When an idea is ready to build, say: `/vault-init [slug]`

Claude will invoke the `projectvault-init-repo` skill, which:
1. Reads the idea's `Design-Spec.md`, `Stack.md`, and `Architecture.md`
2. Generates `ideas/[slug]/ProjectInitialize.md` with full repo bootstrap instructions
3. Updates the idea's status to `Building` in `Overview.md` and `README.md`

Then copy `ideas/[slug]/` → `docs/` in the new empty repo and follow `ProjectInitialize.md`.
